# NestJS 프로젝트 생성 실습

1. NestJS CLI 설치 (프로젝트 생성 도구)
```
$ npm install -g @nestjs/cli
```

2. 프로젝트 생성
```
$ nest new my-backend-app
```

3. 개발 서버 실행 (localhost:3000)
```
$ npm run start:dev
```

### 구조
```
src/app.controller.ts (요청 받는 곳)
src/app.controller.spec.ts (테스트 코드 / 자동 생성됨)
src/app.service.ts (비즈니스 로직)
src/app.module.ts (모듈 등록 -> 앱의 설계도)
src/main.ts (서버 시작점 / bootstrap())
```

- `app.controller.ts`
```ts
import { Controller, Get } from '@nestjs/common';
import { AppService } from './app.service';

@Controller()
export class AppController {
  // 클래스가 만들어질 때 자동으로 실행되는 생성자
  // private : 클래스 내부에서만 사용 
  // readonly : 한 번 값이 들어오면 바꾸지 않음
  constructor(private readonly appService: AppService) {}

  // format : 변수 이름: 변수 타입

  /*
  private readonly appService: AppService;

  constructor(appService: AppService) {
    this.appService = appService;
  }

  와 같지만, NestJS가 직접 의존성 주입한다.
  */

  @Get() // HTTP GET
  getHello(): string {
    return this.appService.getHello();
  }
}

```

- `app.service.ts`
```ts
import { Injectable } from '@nestjs/common';

@Injectable() // 데코레이터, Injectable : 의존성 주입 Dependency Injection
export class AppService { // export : 이 클래스를 다른 파일에게 공개
  getHello(): string { // 함수이름(): 반환타입 {} 
    return 'Hello World!';
  }
}
```

- `app.module.ts`
```ts
import { Module } from '@nestjs/common';
import { AppController } from './app.controller';
import { AppService } from './app.service';

@Module({
  imports: [],
  controllers: [AppController],
  providers: [AppService],
}) 
// {} : 객체 리터럴
// imports : 다른 모듈 가져올때 (ex. AuthModule)
// controllers : 컨트롤러 목록
// providers : 서비스 목록
// -> 전부 NestJS가 관리하고 의존성 주입하도록 등록

export class AppModule {}
```

- `main.ts`
```ts
// main.ts에서 켜진 Nest 서버가 요청 받고 -> Controller -> Service
import { NestFactory } from '@nestjs/core';
import { AppModule } from './app.module';

// NestJS 프로젝트의 실행 시작점
// async : 이 함수 안에서 비동기 작업을 기다릴 수 있다.
async function bootstrap() {
  // NestJS 애플리케이션 객체를 생성하는 NestFactory
  const app = await NestFactory.create(AppModule);
  // NestJS 앱을 만들고 -> AppModule 기준으로 앱 구조를 읽고
  await app.listen(process.env.PORT ?? 3000);
  // -> 서버를 3000 포트에서 실행한다 (process.env.PORT 값이 null (??) 이면 3000 포트에서 실행)
  // await : 이 작업이 끝날 때 까지 기다려라.

  // async : Promise를 반환하는 함수 (지금 당장 결과는 없지만, 나중에 성공/실패할 작업)
  // await : Promise가 끝날 때까지 기다렸다가, 성공 결과를 꺼내라. 
  //         끝나기 전까지 다음 명령은 실행되지 않지만, 다른 요청 처리 등은 가능 (비동기)
}

// bootstrap 함수 실행
bootstrap();
```

## auth 실습
```
$ nest generate resource auth
```
- `nest generate resource`로 관련 모듈 파일을 한번에 만들 수 있다.
- `auth` 라는 기능 단위를 위한 파일을 자동 생성하라는 명령

--- 

- `auth.controller.ts`
```ts
import { Controller, Post, Body } from '@nestjs/common';
import { AuthService } from './auth.service';
import { SignupDto } from './dto/signup.dto';
import { LoginDto } from './dto/login.dto';

@Controller('auth') // 이 컨트롤러의 기본 URL 경로 /auth
export class AuthController {
  constructor( // 의존성 주입
    private readonly authService: AuthService
  ) {}

  @Post('signup')  // POST /auth/signup
  // @Body() HTTP 요청 body를 꺼내오는 데코레이터
  signup(@Body() dto: SignupDto) {
    return this.authService.signup(dto);
  }

  @Post('login')   // POST /auth/login
  login(@Body() dto: LoginDto) {
    return this.authService.login(dto);
  }
}
```

- `auth.service.ts`
```ts
import { Injectable } from '@nestjs/common';
import { SignupDto } from './dto/signup.dto';
import { LoginDto } from './dto/login.dto';

@Injectable() // 이 클래스는 NestJS가 관리 가능한 provider 입니다.
export class AuthService {
  // 실제 DB 대신 메모리 배열 사용 (끄면 사라진다)
  // private ; 클래스 내부에서만 접근 가능
  // 변수명: 타입
  // users: Array<{...}>
  // { id:number, email:string .. }
  // ... = []; 빈 배열
  private users: Array<{
    id: number;
    email: string;
    password: string;
    name: string;
  }> = [];

  signup(dto: SignupDto) {
    const exists = this.users.find(
      (u) => u.email === dto.email
    );
    // (u) => u.email === dto.email
    /*
    이런 느낌과 같다
    function (u) {
        return u.email === dto.email;
    }

    유저 u를 하나씩 꺼내서, u.email === dto.email 인지 검사한다.
    */
    if (exists) {
      return {
        success: false,
        message: '이미 가입된 이메일입니다.',
      };
    }

    const newUser = { id: Date.now(), ...dto };
    // ...dto : dto 객체 안의 필드들을 펼쳐서 넣는다!
    this.users.push(newUser);
    return { success: true, userId: newUser.id };
  }

  login(dto: LoginDto) {
    const user = this.users.find(
      (u) =>
        u.email === dto.email &&
        u.password === dto.password
    );
    if (!user) {
      return {
        success: false,
        message: '이메일 또는 비밀번호가 틀렸습니다.',
      };
    }
    return {
      success: true,
      userId: user.id,
      name: user.name,
    };
  }
}
```

- DTO : (Data Transfer Object)
- `signup.dto.ts`
```ts
export class SignupDto {
  email!: string;
  password!: string;
  name!: string;
}
```
- `email!`의 !는? : Typescript의 definition assignment aseertion. 나중에 들어올테니 지금 constructor에서 초기화하지 않아도 오류 내지 말아라.

- `login.dto.ts`
```ts
export class LoginDto {
  email!: string;
  password!: string;
}
```

### CORS 설정
```ts
// src/main.ts

import { NestFactory } from '@nestjs/core';
import { AppModule } from './app.module';
 
async function bootstrap() {
  const app = await NestFactory.create(AppModule);
 
  app.enableCors({
    origin: 'http://localhost:3001',
  });
 
  await app.listen(3000);
}
bootstrap();
```
- CORS : 교차 출처 리소스 공유 (Cross-origin resource sharing). 출처가 다른 서버 간의 리소스 공유를 허용한다. -> `app.enableCors` 으로, `localhost:3001` (frontend) 에서 오는 요청을 허용한다.
