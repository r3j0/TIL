Operating System Concepts

## Chapter 1 - 서론
운영체제 (Operating System) 는 컴퓨터 하드웨어를 관리하는 소프트웨어이다. 

> - 응용 프로그램을 위한 기반 제공
> - 컴퓨터 사용자와 컴퓨터 하드웨어 사이 중재자 역할 수행
> - 광범위한 컴퓨팅 환경에서 사용 가능 ( 스마트폰, 대형 컴퓨터, 클라우드 컴퓨팅 환경 )
> - CPU, 메모리, 입출력 장치, 저장장치같은 자원들을 프로그램에 할당
> - 컴퓨터 시스템이 동작할 때 이들 자원을 적절하게 사용할 수 있는 방법 제공

### 컴퓨터 시스템 구성 요소
1. 하드웨어
> 중앙 처리 장치(CPU), 메모리 및 입출력(I/O) 장치로 구성. `기본 계산용 자원 제공`
2. 운영체제
> - 다양한 사용자를 위해 다양한 응용 프로그램 간의 하드웨어 사용을 제어하고 조정
> - 다른 프로그램이 유용한 작업을 할 수 있는 환경 제공
3. 응용 프로그램
> 사용자의 계산 문제를 해결하기 위해 자원이 어떻게 사용될지 정의함. ( 워드 프로세서, 스프레드시트, 컴파일러, 웹 브라우저 )
4. 사용자

하드웨어, 소프트웨어 및 데이터로 구성되어 있다고 볼 수 있다.

### 운영체제 역할을 바라보는 관점
1. 사용자 관점 ( User View )
> 컴퓨터에 대한 사용자의 관점은 사용되는 인터페이스에 따라 달라진다.
> - PC, 랩톱 : 한 사용자가 시스템 자원을 독점하도록 설계되었으며, 사용자가 수행하는 작업을 최대화시킴 -> `사용의 용이성을 위해 설계` / 하드웨어와 소프트웨어 자원이 어떻게 공유되느냐의 자원의 이용에는 신경쓰지 않음.
> - 모바일, 셀룰러, 무선 네트워크 기술, 터치스크린, 음성 인식 인터페이스
> - 내장형 컴퓨터 ( 가전제품, 자동차 ) : 사용자의 개입 없이 작동하도록 설계됨

2. 시스템 관점 ( System View )
> 컴퓨터의 관점에서 운영체제는 하드웨어와 가장 밀접하게 연관된 프로그램이다. ( 자원 할당자 (resource allocator) )
> - CPU 시간, 메모리 공간, 저장장치 공간, 입출력 장치 등 문제 해결을 위한 HW, SW 자원들을 효율적으로 공정하게 운영하도록 어느 요청에 자원을 할당할지 결정함.
> - `제어 프로그램 (control program)` 으로써 여러 가지 입출력 장치와 사용자 프로그램을 제어한다. 입출력 장치의 제어와 작동에 깊이 관여한다.

- 커널 : 운영체제가 컴퓨터에서 항상 실행되는 프로그램
- 시스템 프로그램 : 운영체제와 관련되어 있지만 반드시 커널의 일부일 필요는 없는 프로그램
- 응용 프로그램 : 시스템 작동과 관련되지 않은 모든 프로그램
- 미들웨어 : 모바일 운영체제에 종종 핵심 커널과 함께 포함됨. 응용 프로그램 개발자에게 추가 서비스를 제공하는 일련의 소프트웨어 프레임워크

운영체제에는 항상 실행중인 `커널`, 응용 프로그램 개발을 쉽게 하고 기능을 제공하는 `미들웨어 프레임워크`, 시스템 실행 중 시스템 관리에 도움이 되는 `시스템 프로그램`이 포함됨.
